## 프로토콜이란?

컴퓨터끼리 통신할 때 **"어떤 규칙으로 데이터를 주고받을지" 약속한 것.**

사람으로 치면 언어야. 한국어로 말하면 한국어로 답해야 하듯이, FTP로 요청하면 FTP 규칙으로 응답해야 함.

---

## FTP (File Transfer Protocol) — 1971년

파일 전송 전용 프로토콜. 인터넷 초창기부터 있던 방식.

```
클라이언트 ──── 파일 전송 ────▶ 서버
           ◀─── 응답 ──────────
```

- 암호화 없음 → 중간에 누가 패킷 가로채면 내용 그대로 보임
- FileZilla 같은 GUI 프로그램으로 많이 씀
- 서버 접속 후 파일 드래그앤드롭으로 전송

```
[FileZilla]
┌─────────────┬─────────────┐
│  내 PC      │  원격 서버     │
│  app.jar    │  /app/      │
│      ───────────▶         │
└─────────────┴─────────────┘
```

**요즘은 거의 안 씀** — 암호화가 없어서 보안 취약.

---

## SFTP (SSH File Transfer Protocol)

FTP에 암호화를 추가한 버전. SSH 위에서 동작.

```
클라이언트 ════ 암호화된 전송 ═══▶ 서버
```

- FTP랑 사용법은 비슷하지만 데이터가 암호화됨
- FileZilla에서 SFTP도 지원
- "FTP 쓴다"고 하면 요즘은 사실 이걸 쓰는 경우가 많음

---

## SCP (Secure Copy Protocol)

SSH 기반으로 파일을 복사하는 **터미널 명령어**.

```bash
# 로컬 → 서버로 전송
scp app.jar user@192.168.1.100:/app/app.jar

# 서버 → 로컬로 가져오기
scp user@192.168.1.100:/app/app.jar ./app.jar
```

- GUI 없이 터미널 명령어 한 줄로 전송
- SSH 암호화 기반이라 안전
- 파일 전송만 가능 (서버 디렉토리 탐색 불가)

---

## SSH (Secure Shell)

원격 서버에 **터미널로 접속**하는 프로토콜. SCP, SFTP의 기반.

```
클라이언트 ══════════════════════▶ 서버
                                  $ 명령어 실행 가능
```

- 서버에 접속해서 직접 명령어 실행
- 파일 전송(SCP, SFTP)이 SSH 위에서 동작
- 기본 포트: 22번

---

## RDP (Remote Desktop Protocol)

서버 **화면 자체를 원격으로 보여주는** 프로토콜. Windows에서 주로 사용.

```
클라이언트 ══════════════════════▶ 서버
          ◀══ 서버 화면 스트리밍 ══

마우스/키보드 입력 ══════════════▶ 서버에서 실행
```

- 서버 화면이 내 모니터에 그대로 보임
- 마우스, 키보드 입력이 서버로 전달됨
- 파일 복사/붙여넣기도 가능
- GUI가 있어서 가장 직관적

---

## 한눈에 비교

| 프로토콜 | 암호화 | 방식 | 용도 | 비유 |
|---------|--------|------|------|------|
| FTP | ❌ | GUI/명령어 | 파일 전송 | 봉투 없는 우편 |
| SFTP | ✅ | GUI/명령어 | 파일 전송 | 봉투 있는 등기 우편 |
| SCP | ✅ | 터미널 명령어 | 파일 복사 | 택배 한 박스 보내기 |
| SSH | ✅ | 터미널 | 원격 명령 실행 | 전화로 서버한테 명령 |
| RDP | ✅ | GUI | 원격 화면 접속 | 서버 앞에 직접 앉기 |

---

## 배포 방식과의 연관

```
구시대 배포                    현대 배포
─────────────────────────────────────────────
RDP로 서버 접속                SSH로 서버 접속
파일 직접 붙여넣기              git pull / docker pull
수동 재시작                    CI/CD 자동 재시작
```

핵심 차이: **사람이 직접 서버에 들어가서 작업하냐** vs **자동화해서 사람 손이 안 닿냐**
